# TODO visual del portafolio

Lista corta y priorizada de mejoras visuales que harían que el portfolio se vea más profesional, más claro y más atractivo para quien entra al repositorio o navega la web.

## Prioridad alta

- Agregar una captura o mockup grande al inicio del README y del portfolio para mostrar el resultado final apenas se abre.
- Unificar mejor la jerarquía visual entre secciones, con más contraste entre títulos, subtítulos y contenido.
- Darle más aire a los bloques principales para que el contenido no se sienta tan cargado.
- Mejorar las tarjetas de experiencia, cursos y proyectos para que tengan una línea visual más consistente entre sí.
- Resaltar más el CTA principal de contacto y CV con una composición más fuerte en el hero.

## Prioridad media

- Replantear la paleta de colores para que tenga un acento más distintivo y menos repetido en toda la interfaz.
- Mejorar los estados hover y active para que sean más elegantes y menos genéricos.
- Ajustar tipografías, tamaños y espaciados para que la lectura se sienta más premium.
- Agregar pequeños separadores visuales entre secciones para mejorar el ritmo de scroll.
- Hacer que los badges, chips y etiquetas tengan un estilo más pulido y uniforme.

## Prioridad baja

- Sumar una sección visual de métricas o highlights arriba del todo.
- Agregar animaciones de entrada más suaves y menos repetitivas.
- Mejorar la presentación del README con badges, captura y una mini galería.
- Revisar el pie de página para que cierre mejor la identidad visual del sitio.

## Ideas extra

- Agregar una imagen de preview para GitHub con el portfolio completo.
- Crear una versión visualmente más editorial para la sección de experiencia.
- Introducir un patrón sutil de fondo o textura para evitar superficies demasiado planas.
